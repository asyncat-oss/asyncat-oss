# Stakeholder Updates

## Audience Analysis

### Executive
- Focus on business impact
- High-level status
- Risks and decisions

### Client
- Progress focused
- Clear deliverables
- Timeline and budget

### Team
- Technical details ok
- Blockers important
- Action items

## Update Structure

### 1. Executive Summary
- 2-3 sentences
- Overall status (Green/Yellow/Red)

### 2. Key Highlights
- Top 3 wins
- Major milestones

### 3. Challenges
- Current blockers
- Risks to watch
- Needed decisions

### 4. Next Steps
- Upcoming milestones
- Timeline changes

### 5. Metrics (if applicable)
- KPIs
- Progress %
- Budget status

## Tone

### Positive but Realistic
- Acknowledge issues
- Don't sugarcoat
- Focus on solutions

### Concise
- Bullets over paragraphs
- Data over words
- One page max

## Frequency

### Weekly
- Quick status
- Blockers only

### Bi-weekly
- Progress + risks
- Metrics optional

### Monthly
- Full update
- Presentation format

## Template

```
PROJECT UPDATE: [Project Name]
Date: [Date]
Period: [Date Range]

📊 STATUS: 🟢 Green / 🟡 Yellow / 🔴 Red

✅ HIGHLIGHTS
- [Win 1]
- [Win 2]

⚠️ CHALLENGES
- [Challenge 1]
- [Challenge 2]

📅 LOOKAHEAD
- [Upcoming milestone 1]
- [Upcoming milestone 2]

🎯 ACTION NEEDED
- [Decision required]
```

## Best Practices
- Be consistent with format
- Be honest about issues
- Always provide next steps
